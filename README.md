# 제니스 랜딩 운영 소스

제니스 웹솔루션에서 생성한 이벤트 랜딩 페이지를 외부 도메인에서 운영하기 위한 PHP 런타임 모듈입니다. 이벤트 번호 기반 URL로 랜딩 파일을 렌더링하고, 신청 데이터 수집부터 유효성 검증, 중복/블랙리스트 처리, 외부 제휴사 연동, 재전송 관리까지 랜딩 운영에 필요한 흐름을 담당합니다.

관리자 시스템이 이벤트, 광고주, 매체, 스크립트, 연동 조건을 관리하고 이 운영 소스는 실제 방문자 요청을 받아 랜딩 페이지와 신청 처리 로직을 실행하는 구조입니다. 마케팅 캠페인 운영 중 발생하는 대량 트래픽과 다양한 제휴사 전송 포맷을 처리할 수 있도록 단순한 PHP 파일 기반으로 구성했습니다.

## 주요 기능

- 이벤트 번호 또는 해시 URL 기반 랜딩 페이지 라우팅
- 랜딩별 HTML, 처리 파일, 완료 페이지 동적 로드
- 고객 신청 데이터 저장 및 신청 완료 리다이렉트
- 이름, 연락처, 나이, 성별, 쿠키, IP 기반 중복/불량 데이터 검증
- 블랙리스트 연락처 차단 및 신청 상태 자동 분류
- Facebook Pixel 전환 이벤트 전송
- 외부 제휴사 연동 결과 저장 및 실패 건 재처리
- 방문/신청 요청 로그 저장
- 내부 IP 기준 서버 정보 확인 및 운영용 디버그 접근 제어

## 처리 흐름

1. 방문자가 이벤트 URL로 접근합니다.
2. `zenith_core.php`가 이벤트 번호를 해석하고 DB에서 랜딩 정보를 조회합니다.
3. 랜딩 파일을 로드하고 방문자 식별 쿠키와 세션 값을 설정합니다.
4. 신청 데이터가 전송되면 `CheckProc`가 필수값, 중복, 블랙리스트, 불량 조건을 검증합니다.
5. 정상 신청은 DB에 저장하고 완료 페이지로 이동합니다.
6. 연동 대상 이벤트는 제휴사 API로 신청 데이터를 전송하고 응답을 기록합니다.
7. 실패한 연동 건은 재전송 스크립트로 후속 처리합니다.

## 소스 구성

| 파일 | 설명 |
| --- | --- |
| `zenith_core.php` | 랜딩 라우팅, 페이지 렌더링, 신청 처리, 로그 저장, 픽셀 전송 |
| `zenith_check_proc.php` | 신청 데이터 검증, 중복 체크, 상태 분류, 저장 후 처리 |
| `zenith_interlock.php` | 외부 제휴사 연동 결과 저장 및 실패 건 재처리 |
| `zenith_ip.php` | 방문자 IP 판별, 내부 IP 체크, 반복 접근 차단 |
| `zenith_encryption.php` | 연락처 및 완료 파라미터 암복호화 |
| `zenith_cookie.php` | 랜딩 운영용 쿠키 헬퍼 |
| `interlock_resend.php` | 특정 실패 연동 건 재전송 |
| `interlock_resend_all.php` | 조건 기반 대량 연동 재전송 |
| `zenith_test.php` | 연동 테스트 요청 |
| `zenith_test_form.php` | 신청 테스트 폼 |

위처럼 일부 소스파일만 공개

## 실행 환경

```bash
cp .env.example ../.env
```

`../.env`에는 암복호화 키를 설정합니다. DB 연결과 쿼리 메서드는 `zenith_db.php`의 `ZenithDB` 클래스가 담당합니다.

필수 런타임 구성은 다음과 같습니다.

- PHP 웹 서버
- MySQL 접근 권한
- `openssl`, `sodium`, `curl`, `mbstring` 확장
- 랜딩 파일 디렉터리 `data/`
- 공통 레이아웃 디렉터리 `inc/`
- 에러 페이지 디렉터리 `error/`
